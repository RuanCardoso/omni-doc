    Getting Started¶

    Installation¶

    Install necessary dependencies before starting your project.

    com.unity.nuget.newtonsoft-json

    How to install newtonsoft-json?

    1. Open the Unity Editor.
    2. Go to Window > Package Manager.
    3. Click on the + button in the top-left corner of the Package Manager window.
    4. Press Add package by name.
    5. Enter com.unity.nuget.newtonsoft-json as the package name and press Add or Enter.
    6. Wait for the installation to complete.

    Note

    Before installing check if the package is already installed. some versions of Unity include the package com.unity.nuget.newtonsoft-json by default. By default, Omni uses binary serialization, but it can use JSON serialization as well.

    • Newtonsoft.Json is installed.

    Install Omni:

    1. Go to the GitHub repository Releases and download the latest version.
    2. Go to Assets > Import Package > Custom Package and select the downloaded version and press Import.
    3. Wait for the installation to complete and scripts recompilation.

    Note

    After installation, note that the following macros will be automatically defined in your project:

    • OMNI_DEBUG Automatically activated for debugging purposes. This macro is enabled by default.
    • OMNI_RELEASE Should be used for production builds. This activates all possible optimizations.
    • OMNI_SERVER Automatically activated when building the project for server deployment (Dedicated Server mode).

    Omni most used namespaces:

    • using Omni.Core; Used to all networking related classes, including database classes.
    • using Omni.Core.Hashing; Provides hashing utilities.
    • using Omni.Core.Web; Provides web functionality, like as, Express.js, Web Server, HTTP Request, etc.
    • using Omni.Core.IMatchmaking; Provides matchmaking functionality, like as, Rooms, Groups, etc.
    • using Omni.Execution; Provides database query execution.
    • Omni is installed.

    Wow!, now you can start your project

    Setup Omni¶

    1. Go to Omni in the Unity Menu, next to the Window menu and press Setup
    2. Go to the Hierarchy tab, select and expand the Omni Networking game obect.

    Let's take a look at what each of these scripts and configurations are now.

    Tip

    You can skip this step if you want, as Omni comes pre-configured. Play with the default settings and see what happens.

    Property Description
    Production Mode Enables or disables production mode. Production mode should be enabled when you are ready to deploy your game, ensuring a stable and optimized environment for end-users.
    Guid A unique identifier for the current game version. If the server's GUID differs from the client's GUID, the connection will fail, ensuring consistency and integrity between client and server versions. Omni Network Script > Context Menu > Generate GUID
    Physics Mode Defines the physics mode to be used. Use 2D mode for two-dimensional games and 3D mode for three-dimensional games, ensuring accurate simulation of physical behavior according to your game's needs. This option is only used to simulate physics in scenes automatically created by Omni.
    Loop Mode Specifies the loop mode to be utilized. Choose between Tick-Based or Realtime, determining the execution model for game updates. Tick-Based provides a consistent time interval, while Realtime operates according to the framerate.

    Warning

    Tick-Based update method: 

    // you must inherit from `NetworkBehaviour` or `RealtimeTickBasedSystem`. inherit from `NetworkBehaviour` is recommended.
// these methods are not called in realtime mode.
public class TickExample : NetworkBehaviour
{
    public override void OnUpdateTick(ITickData tick)
    {
        // is called every tick
    }

    public override void OnUpdateTickStart(ITickData tick)
    {
        // is called on the first tick
    }

    public override void OnUpdateTickEnd(ITickData tick)
    {
        // is called on the last tick
    }
}

    Realtime update method: 

    // you must inherit from `NetworkBehaviour` or `RealtimeTickBasedSystem`. inherit from `NetworkBehaviour` is recommended.
public class TickExample : NetworkBehaviour
{
    // this method is not called in tick-based mode.
    public override void OnUpdate()
    {
        // is called every frame
        // the same as Update() which is called by Unity.
    }

    // or independent, called by Unity.
    public void Update()
    {
        // is called every frame
    }
}

    Property Description
    Connect Async Initiates an asynchronous connection to the server. This method allows for non-blocking connection establishment, ideal for scenarios where responsiveness is crucial.
    Transport Option Specifies the transport protocol utilized for communication with the server. Options include TCP, UDP or Web Sockets. UDP(Lite) is the default.

    Warning

    TCP Layer Availability: TCP transport supports the Reliable and reliable encrypted layers. If other layers are specified, they will be ignored, and the default Reliable layer will be utilized.

    WebSocket Layer Availability: Web-Socket transport supports the Reliable and reliable encrypted layers. Specifying other layers will be disregarded, and the default Reliable layer will be employed. Encryption can be enabled by installing an SSL certificate.

    UDP Layer Availability: UDP transport offers support for Reliable, Unreliable, and reliable encrypted layers. All layers are accessible for UDP transport. Encryption is provided through default AES and RSA encryption implementations.

    Property Description
    IOPS Increases the socket's receive rate to match higher transmission rates. A higher value should be set to ensure the read rate keeps pace with the transmission, preventing bottlenecks and ensuring smooth data flow.
    FPS Update Rate Specifies the frames per second (FPS) update rate provided by the network monitor. This value indicates the frequency of FPS updates.
    OnTransportSettings Called before Omni starts, allowing for configuration adjustments. Use this to modify Omni settings before initialization, ideally used for setting platform-specific configurations.

    Tip

    Example of OnTransportSettings: 

    // inherit from `CustomTransportSettings`
public class CustomSettings : CustomTransportSettings
{
    public override void OnTransportSettings(TransportSettings transportSettings, RuntimePlatform runtimePlatform)
    {
        if(runtimePlatform == RuntimePlatform.Android)
        {
            transportSettings.Host = "89.0.142.86"; // example
        }
    }
}

    Attach this script to your game object, then drag the game object into the OnTransportSettings field.

    Warning

    Some options below are transporter-specific, which means certain options are exclusive to transporter X and are not available for transporter Y, but if set they will be ignored.

    Property Description
    Use Native Sockets Enables or disables the use of native socket libraries. This property is exclusively available on Windows and Linux platforms.
    Enable IPV6 Enables the usage of IPV6 protocol for network communication.
    Use Safe MTU Automatically determines the optimal Maximum Transmission Unit (MTU) to reduce packet loss and ensure reliable network communication. This feature adjusts the MTU size to mitigate the risk of packet fragmentation and loss during data transmission.
    Disconnect Timeout Sets the duration in milliseconds after which the client is automatically disconnected if no packets are received.
    Packet Pool Size Determines the number of packets available in the packet pool for network communication.
    Ping Interval Specifies the time interval, in milliseconds, between ping requests to the server for checking the connection status. Longer intervals may reduce network overhead, while shorter intervals offer more frequent connection monitoring.
    No Delay Activates the TCP No Delay option, reducing latency by disabling Nagle's algorithm. This option can be beneficial for applications requiring immediate data transmission without delay.
    Linger State Specifies the linger state of the socket. The linger state determines whether the socket will continue to linger after closing, allowing pending data transmissions to complete.
    Send Buffer Size Specifies the size of the send buffer, optimizing the capacity for outgoing data transmissions.
    Receive Buffer Size Determines the size of the receive buffer, optimizing the capacity for incoming data transmissions.
    Send Timeout Sets the duration for the send operation to complete. If the send operation exceeds this duration, it will be aborted.
    Receive Timeout Specifies the duration for the receive operation to complete. If the receive operation exceeds this duration, it will be aborted, indicating a timeout error.

    Warning

    The options below are available for all transporters but may be ignored if set unnecessarily.

    Property Description
    Host Specifies the hostname or IP address of the server. This value is used to establish the connection to the server.
    Server Port Defines the port number used by the server. This value is essential for establishing the connection to the server.
    Max FPS Sets the maximum frames per second (FPS) for the client. This option does not affect server builds.
    Max Message Size Specifies the maximum size of messages that the server can receive. This limit ensures efficient handling of data and prevents overload.
    Max Connections Determines the maximum number of connections that the server can accept concurrently. This limit helps manage server resources effectively.

    Web-Server Configuration¶

    Property Description
    Enable Web Server Activates the usage of the web server, facilitating web-based interactions with the application.
    Web Server Port Specifies the port number for the web server, defining the communication channel for web-based interactions.
    Use Https Enables Secure Sockets Layer (SSL) encryption for secure communication over HTTPS, enhancing data security and integrity.
    Process PHP Enables the processing of PHP files by the web server, allowing for dynamic content generation and enhanced functionality within web applications.
